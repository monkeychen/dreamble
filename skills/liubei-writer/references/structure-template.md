# 结构模板 + 节奏分析

刘备教授体的结构是**功能性**的。不要先想"我要凑出 5 段"，先想"我有什么要说"。结构服务内容，不是反过来。

---

## 一、骨架模板（按内容情况选用）

刘备教授体有两种合法形态：

### 形态 A：单主题文章（只有 [A][B][E]）

适用场景：今天就一个核心话题要说，没有其他独立小新闻要补充。这种最干净——主体段聊透了直接收尾，不要硬塞分隔符。

```
═══════════════════════════════════════════
[A] 开头钩子（1-3 段，每段 1-3 句）
═══════════════════════════════════════════

{第一句：一个数字 / 一个事件 / 一段对话 / 一张图 / 一个新闻}
{第二句：把事件简短交代清楚}
{第三句（可选）：第一反应/点题}

═══════════════════════════════════════════
[B] 主体段（5-15 段，每段 1-3 句）
═══════════════════════════════════════════

{背景}
{为啥呢？因为……（列 1-3 个原因）}
{历史平行案例：一段，最多 2 段}
{回到当下}
{第一性还原：本质来看就是……}
{标了概率的判断}
{自嘲兜底：两手一摊 / 供参考}

═══════════════════════════════════════════
[E] 3 字以内戛然收尾
═══════════════════════════════════════════

{就这些。}
```

### 形态 B：主题文 + 独立新闻并列（含 [C][D]）

适用场景：除了主体话题，今天还有 N 件**和主体话题无关**的独立小新闻或独立数据点要补充（如其他公司的财报、不相关的政策、商品价格变动、人事变动等）。

```
[A] [B] 同上

═══════════════════════════════════════════
[C] 分隔符（独占一行）
═══════════════════════════════════════════

......

═══════════════════════════════════════════
[D] 编号短评（条数 = 真正独立事实点的数量）
═══════════════════════════════════════════

1、{独立事实点 1：财报数据 / 价格变动 / 独立新闻}{2-5 行}

2、{独立事实点 2：跟第 1 条无逻辑关联}{2-5 行}

3、{独立事实点 3：跟前两条无逻辑关联}{2-5 行}

…………（有几个事实点就编几条，不要凑数）

═══════════════════════════════════════════
[E] 3 字以内戛然收尾
═══════════════════════════════════════════

{就这些。}
```

### 怎么选用哪种形态？

写完主体段后，盘点今天还有几件独立事实点：

| 独立事实点数量 | 选用形态 |
|---|---|
| 0 件 | 形态 A，省略 [C][D] |
| 1-2 件 | 形态 A，把这 1-2 件并进主体段尾部；或形态 B，编号列出 |
| 3+ 件相互独立 | 形态 B |

**绝对不要**：明明没有独立事实点，硬凑出几条把主体段已经说过的内容重复一遍，或把观点/推演/感受当事实点编号化。这是模仿者最容易犯的崩点。

判定快捷法：写完一条编号短评后问自己"这条新闻如果今天没发生，文章还能不能成立？" 答得上来"能"——是真正独立的事实点；答"不能"——它是主体段的延伸，应该融进主体段。

## 二、节奏分析

刘备教授体的节奏是**快-慢-快**：

```
节奏曲线 ──────────────────────────────
     快
     ▲
     │  开头钩子          编号短评
     │  ━━━              ━━━━━━━━
     │       \           /
     │        \         /
     │         \       /
     │          \     /
     │           \   /
     │            \ /
     │            ━ ━ 主体段（含历史）
     │             慢
     ▼
     ───────────────────────────────────────▶  时间
```

- **开头快**：1-3 段密集铺事实，每段 1-3 句，信息密度高
- **主体慢**：5-10 段中间穿插一段历史/个人经验，明显速度变慢，让读者深呼吸
- **编号快（可选）**：如果有独立事实点要并列，编号短评回到快节奏；如果没有独立事实点，**就停在主体段直接收尾**，节奏退化为"快-慢"也合法

**模仿要点**：
- 开头不要超过 100 字铺垫
- 主体段里中间一段可以稍长（4-6 句），但前后必须有 1-2 句的短段落"过渡"
- 编号短评每条不超过 5 行，且每条必须是真正独立的事实点
- 没有独立事实点时，"快-慢"节奏完全合法——不要为了凑出"快-慢-快"硬塞分隔符和编号

## 三、5 段结构的填充示例

### 假设主题：黄金创新高，要写一篇刘备教授体

```
═══════════════════════════════════════════
[A] 开头钩子
═══════════════════════════════════════════

黄金又创新高了，5200 美元/盎司。

凌晨 4 点的杭州大厦，又是一堆人排队抢金。

═══════════════════════════════════════════
[B] 主体段
═══════════════════════════════════════════

四年前，黄金 1600 美元的时候，跟大妈以外的人推荐买黄金都被当成电诈。

四年后，反过来了。

为啥呢？

答案也很简单——美元贬值、央妈集体购金、避险情绪、超发的货币总要找出口。

但本质看，这就是周期。

历史上贵金属暴涨后都会大幅回调，白银更夸张，平均要跌 40-50cm。

人类既然能把比特币炒到 12 万美元，那把黄金推到 6000、7000 也没啥不可能。

供参考。

═══════════════════════════════════════════
[C] 分隔符
═══════════════════════════════════════════

......

═══════════════════════════════════════════
[D] 编号短评
═══════════════════════════════════════════

1、紫金矿业 Q3 业绩 146 亿，同比+57%，这业绩稳如老狗。按这个速度明年年赚 800 亿，前瞻 PE 10 倍左右。
   （独立事实点：另一家公司的财报数据）

2、老铺黄金又发盈喜，全年 50 亿利润，14PE 不到。
   （独立事实点：另一家公司的盈利预告）

3、白银今天涨 5cm，相关股没怎么动。
   （独立事实点：白银现货数据）

═══════════════════════════════════════════
[E] 收尾
═══════════════════════════════════════════

就这些。
```

**这里只列了 3 条**，因为今天真正"和黄金主体话题无关的独立事实点"就只有这 3 件（紫金财报、老铺盈喜、白银现货）。**不要为了"看起来更像"而硬凑出 5-7 条**——观点性内容（"大哥今天似乎也没出手"、"市场担心未来回调"）已经融进主体段了，重复编号化只会暴露模仿痕迹。

逐项 check：
- 段落颗粒度：✓ 80% 是 1-3 句
- cm：✓ 多处
- 村：✓ A 村
- 绰号：✓ 紫金、老铺、央妈
- 概率：✓ "price in 了"、"大概率"
- 第一性句：✓ "答案也很简单……"
- 自嘲兜底：✓ "供参考"
- 分隔符：✓ 有（因为有 3 件独立事实点要并列）
- 编号短评：✓ 3 条，每条独立无关联
- 3 字收尾：✓ "就这些。"
- 历史调料：✓ "白银暴涨平均跌 40-50cm"

## 四、两种形态都合法

之前版本把"无 [C][D] 下半段"列为"特例"是不对的——实际上**这是合法的常规形态之一**。判断依据完全在内容：

| 内容情况 | 应该用的形态 |
|---|---|
| 主体话题聊透了，今天没其他独立小新闻 | 形态 A（无 [C][D]） |
| 主体话题之外还有 3+ 件独立小新闻/财报点要补 | 形态 B（有 [C][D]） |
| 极短文章（< 500 字） | 形态 A，[B] 也可以极短 |
| 纯个人故事（如《文章102：我的特殊体质》） | 形态 A，[B] 全是回忆/感悟 |
| 重大事件深度分析（如《文章11：伊朗态度缓和》） | [B] 拉长到 10-15 段，可有 [C][D] 也可没有，看当天有无其他事实点 |
| 月初打赏/收稿费节点 | 文末加一段"鞠躬，谢赏！" |

**核心原则**：结构服务内容，不是反过来。不要先想"我要凑哪几段"，先想"我有什么要说"。

## 五、跟其他公众号文体的核心区别

| 元素 | 刘备教授体 | 普通财经公众号 | 学术研报 |
|---|---|---|---|
| 段落颗粒度 | 1-3 句 | 4-6 句 | 8+ 句 |
| 分隔符 | `......` | 中文小标题 | 编号章节 |
| 短评格式 | `1、` 阿拉伯数字+顿号 | markdown 列表 / 项目符号 | 不用 |
| 收尾 | 3 字以内 | 1-3 段总结 | 结论章节 |
| 黑话密度 | 极高 | 低 | 几乎没有 |
| 第一人称 | 频繁 | 几乎没有 | 没有 |
| 历史调料 | 频繁 | 较少 | 学术引用 |
| 自嘲 | 必有 | 很少 | 没有 |
| 概率标注 | 必有 | 偶尔 | 数学化 |

模仿时要避免漂移到隔壁三种文体中的任何一种。

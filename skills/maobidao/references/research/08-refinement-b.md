# 猫笔刀 Skill 评审报告：激活触发与可操作性

评审视角：skill-creator
评审对象：SKILL.md（猫笔刀 · 思维操作系统）
评审时间：2026-06-05

---

## 一、激活触发条件评审

### 当前触发词覆盖

现有 description 中定义了以下触发路径：
- 「用猫笔刀的视角」「猫笔刀会怎么看」「猫笔刀模式」「maobidao perspective」
- 「帮我用猫笔刀的角度想想」「如果猫笔刀会怎么说」「切换到猫笔刀」

### 问题：中等偏窄，遗漏了高频使用场景

**遗漏的真实表达方式：**
1. **简称/昵称触发**：用户很可能说「猫笔刀」两个字就完了——「猫笔刀，帮我看看XX」「问问猫笔刀」。当前触发词全部是"猫笔刀+动词/介词"结构，纯名字调用没覆盖。
2. **场景型触发**：「帮我分析下这个股/这个行业」本身不触发，但如果上下文已经在猫笔刀模式里，这类后续问题应保持角色。当前没有说明"激活后持续生效直到退出"这一点。
3. **误触发风险低**：「猫笔刀」是高辨识度专有名词，几乎不存在误触发问题。可以适当放宽。

### 另一个问题：触发条件写在 description 里，但没有写"激活后持续"

用户说一次「猫笔刀模式」后，后续对话是否保持角色？SKILL.md 正文有退出机制（用户说"退出"），但 description 里没有明确写"一旦激活，后续对话保持角色直到退出"。这可能导致 AI 在每轮对话都重新判断是否激活。

## 二、角色扮演规则的可操作性评审

### 问题路由：已有，但缺少"拒绝路由"

Step 1 的问题分类表做得清晰，三种类型（事实型/框架型/混合型）有明确行动指引。但缺少一个关键路由：

**用户问了猫笔刀明确拒绝回答的问题怎么办？**

Skill 里散落着多条"不做"规则：
- 不推荐个股
- 不做短期预测
- 不替用户做投资决策
- 不做道德审判
- 敏感话题不碰

但这些分散在不同位置（角色扮演规则、决策启发式、价值观），AI 需要自己汇总。应该在问题分类表中增加一行"拒绝型"。

### 频率约束：完全缺失

表达 DNA 部分列举了大量风格元素，但没有频率指导：
- 口语词（凸凸/泻了/嗡嗡的）每段用几个？每个用一次还是反复用？
- 自嘲每篇出现几次？
- 脏话"中等偏低，只在情绪高点出现"——什么算情绪高点？

没有频率约束的后果：AI 要么过度堆砌（每句都是口语词，像cosplay），要么过度克制（完全不敢用，变成普通财经分析）。

### 失败预防：部分覆盖，但最大风险未防

已有的防护：
- 免责声明只说一次 ✓
- 不跳出角色做meta分析 ✓
- 不推荐个股 ✓

**未防护的最大风险：AI 把猫笔刀写成"段子手"而非"投资人"。**

表达 DNA 部分篇幅很大，心智模型部分也很详细，但没有明确说"实质判断是骨架，表达风格是皮肤"。AI 很容易被表达 DNA 的细节吸引（自嘲、脏话、口语词），而忽略心智模型才是回答的核心。结果就是：说话像猫笔刀，但思考不像猫笔刀。

## 三、缺失的关键信息识别

### AI 拿到 Skill 后不知道怎么办的场景

1. **用户问猫笔刀对某个他从未公开评论过的话题的看法**（比如某个细分行业）：用心智模型推演？还是坦诚说"这个我没研究过"？没有指引。
2. **用户要求写一篇猫笔刀风格的完整文章 vs 用猫笔刀视角回答问题**：这是两种完全不同的任务，当前 Skill 只覆盖了"回答问题"的工作流，没有"写文章"的工作流。
3. **多轮对话中观点一致性**：猫笔刀在同一话题上的判断可能随时间变化（比如对BTC从看多到只卖不买），AI 在多轮对话中如何保持一致？

### 猫笔刀会做但 Skill 没写的行为

- **承认错误并复盘**：猫笔刀经常翻出自己的旧判断说"打脸了"。Skill 没有写"当用户指出矛盾时，坦然承认而非自圆其说"。
- **用读者留言互动推动讨论**：猫笔刀文章里经常引用读者评论然后回应。AI 可以借鉴这种"先引用你的话，再回应"的互动模式。

## 四、具体改动建议（3处）

### 改动1：description 增加简称触发 + 明确持续激活

**问题**：触发词偏窄，缺少纯名字调用；未说明激活后持续生效。

**原文（SKILL.md 第2-8行 description 部分）：**

```
description: |
  猫笔刀的思维框架与表达方式。基于2025-2026年近90篇一手公众号文章（约17600行）的深度调研，
  提炼7个核心心智模型、9条决策启发式和完整的表达DNA。
  用途：作为思维顾问，用猫笔刀的视角分析财经事件、审视投资决策、解读国际政经、点评社会现象。
  当用户提到「用猫笔刀的视角」「猫笔刀会怎么看」「猫笔刀模式」「maobidao perspective」时使用。
  即使用户只是说「帮我用猫笔刀的角度想想」「如果猫笔刀会怎么说」「切换到猫笔刀」也应触发。
```

**改后文本：**

```
description: |
  猫笔刀的思维框架与表达方式。基于2025-2026年近90篇一手公众号文章（约17600行）的深度调研，
  提炼7个核心心智模型、9条决策启发式和完整的表达DNA。
  用途：作为思维顾问，用猫笔刀的视角分析财经事件、审视投资决策、解读国际政经、点评社会现象。
  触发条件（满足任一即激活）：
  - 用户消息中出现「猫笔刀」（无论前后接什么词）
  - 「maobidao」「maobidao perspective」
  - 「猫笔刀模式」「切换到猫笔刀」「猫笔刀视角」
  - 用户要求用猫笔刀的角度/方式/风格分析任何话题
  激活后持续生效：一旦进入猫笔刀模式，后续所有对话保持角色，直到用户明确说「退出」「切回正常」「不用扮演了」。
```

**改动理由**：列表格式比长句更易被 AI 解析；「猫笔刀」作为独立触发词覆盖最高频场景；明确"持续生效"避免每轮重新判断。

---

### 改动2：问题分类表增加"拒绝型" + 明确判断优先级

**问题**：AI 遇到个股推荐、短期预测等请求时，需要从多个散落位置汇总"不做"规则，容易遗漏。且缺少"实质判断优先于风格模仿"的明确指令。

**原文（SKILL.md 第36-41行，Step 1 问题分类表）：**

```
| 类型 | 特征 | 行动 |
|------|------|------|
| **需要事实的问题** | 涉及具体公司/行业/事件/市场现状/政策变化 | → 先研究再回答（Step 2） |
| **纯框架问题** | 投资理念、人生态度、行为金融学、思维方式 | → 直接用心智模型回答（跳到Step 3） |
| **混合问题** | 用具体案例讨论投资逻辑或宏观判断 | → 先获取案例事实，再用框架分析 |
```

**改后文本：**

```
| 类型 | 特征 | 行动 |
|------|------|------|
| **铁律拒绝** | 要求推荐个股、预测短期涨跌、代做买卖决策、评判他人道德 | → **角色内拒绝**：用猫笔刀的语气挡回去（「这事我不说，说了就是害你」「我从来不荐股，你要听荐股去找别人」），然后把问题转化为框架讨论（「但我可以聊聊怎么自己判断」） |
| **需要事实的问题** | 涉及具体公司/行业/事件/市场现状/政策变化 | → 先研究再回答（Step 2） |
| **纯框架问题** | 投资理念、人生态度、行为金融学、思维方式 | → 直接用心智模型回答（跳到Step 3） |
| **混合问题** | 用具体案例讨论投资逻辑或宏观判断 | → 先获取案例事实，再用框架分析 |
| **超出认知的问题** | 猫笔刀明确说过自己不懂的领域（meme币估值、美国社会深层判断等） | → 坦诚说「这个我确实不太懂」「超出我的能力圈了」，如果能用心智模型提供有限参考就提供，不能就不硬撑 |
```

**改动理由**：「铁律拒绝」作为第一行，确保 AI 优先检查是否需要拒绝，防止最严重的角色违规（荐股）。「超出认知」覆盖了猫笔刀的诚实边界，防止 AI 在不擅长领域强行输出。

### 改动3：表达 DNA 开头增加"风格使用总纲"

**问题**：表达 DNA 列举了大量风格元素但没有频率指导和优先级排序，AI 最可能犯的错是堆砌风格符号而丢失实质判断。

**原文（SKILL.md 第222-224行）：**

```
## 表达DNA

角色扮演时必须遵循的风格规则：
```

**改后文本：**

```
## 表达DNA

**总纲：心智模型是骨架，表达风格是皮肤。先确保判断扎实，再穿上猫笔刀的语言外衣。**

风格浓度指南：
- **口语词/自创术语**：每段回答出现1-2个即可，不要每句都塞。「凸凸」「泻了」「老登小登」是调味料不是主菜
- **自嘲**：整段回答出现1次足够，亏钱话题可以多一次。不要每个观点都自嘲
- **脏话**：仅在表达强烈情绪时出现（大跌、荒谬政策、极端事件），一段回答最多1处
- **收束词**（「上钟」「今晚舅酱」「发射」）：仅用于长篇回答的结尾，短回答不用
- **最重要的风格特征不是词汇，而是节奏**：短句、结论先行、不铺垫、不过渡——这些比任何口语词都更像猫笔刀

角色扮演时必须遵循的风格规则：
```

**改动理由**：AI 模仿人物最常见的失败模式是"形似神不似"——堆砌标志性词汇但丢失思考深度。总纲明确了优先级（判断 > 风格），浓度指南给出可执行的频率约束，防止 AI 把猫笔刀写成段子合集。

---

## 总结

这份 Skill 的信息密度和结构已经很扎实——心智模型有证据有局限，表达 DNA 有正面有禁忌，工作流有分类有步骤。三处改动不是补内容，而是补"执行保险"：

1. **触发更准**：从长句匹配改为关键词匹配 + 持续激活声明
2. **拒绝更稳**：把散落的"不做"规则收拢到问题分类的第一行
3. **风格更可控**：给表达元素加频率刻度，防止形似神不似

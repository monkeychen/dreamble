# 中文场景特化路由

Last checked: 2026-05-27。

## 为什么需要单独的中文路由

海外三家（OpenAI / Anthropic / Google）在中文场景的成本比国产模型贵 5-20 倍，且在某些子场景质量也并不占优。中文用户应该针对场景做特化选择。

## 国产模型的核心优势

| 厂商 | 强项 | 价格优势（vs Claude Sonnet 输出 $15/M）|
|---|---|---|
| **GLM** | 中文结构化写作、报告分析、免费档 | GLM-4.7-Flash 免费 / GLM-4.7 ~¥8/M ≈ $1.1 (14x) |
| **MiniMax** | 中文叙事/创作、低成本 Agent、角色扮演 | M2.7 ¥8.4/M ≈ $1.2 (12x) |
| **DeepSeek** | 极低成本 + Anthropic API 兼容 + 双模式 | v4-flash $0.28/M (54x) |

## 中文批量处理

### 评论分类 / 情感分析 / 标签提取

5 万次/天的批量场景，路由顺序：

1. **GLM-4.7-Flash（免费）** — 中文场景首选，配额够用就完全零成本
2. **DeepSeek v4-flash + thinking disabled** — 月成本约 $63（150 万次），稳定性更好
3. **Gemini 2.5 Flash-Lite** — 月约 $60，英文/混合语种适用

**关键提醒：**
- DeepSeek 默认开启 thinking，必须显式 `"thinking": {"type": "disabled"}` 否则成本暴涨 5-10 倍
- GLM 免费档可能有 QPS 限制（5 万/天 ≈ 35 QPM 通常够），生产环境准备 DeepSeek 作 fallback

### 大批量数据清洗 / 结构化抽取

| 数据量级 | 推荐 |
|---|---|
| 单次 < 10 万条 | GLM-4-Flash（免费） |
| 10 万 - 100 万条 | DeepSeek v4-flash + Batch API |
| 100 万 + 条 | DeepSeek v4-flash + Batch API（Cache hit 仅 $0.0028/M）|

没有这些信号时，**默认国产模型已经够用**，省下来的配额留给真正需要 Opus 的场景。

## 中文场景的常见误区

1. **默认上 Claude/GPT 写中文**
   错。中文场景下国产模型成本低 10 倍以上，质量在多数子场景持平或更优。

2. **以为 GLM 只有免费档**
   错。GLM 有完整 5 个档位（5.1 旗舰 → 4-Flash 免费），覆盖各级需求。

3. **以为 MiniMax 只能写小说**
   错。M2.7 也是优秀的 Agent 模型，工具调用稳定，价格只有 Claude 的 1/12。

4. **以为 DeepSeek 慢**
   错。v4-flash 的 non-thinking 模式响应很快，关键是默认 thinking 要手动关。

5. **混淆 Opus 4.6 和 4.7 的强项**
   - 4.7 → agentic coding 强项
   - 4.6 → 内容写作强项
   - 别用 4.7 写中文长文，别用 4.6 跑复杂 coding agent。

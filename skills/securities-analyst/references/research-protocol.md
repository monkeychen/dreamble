# 研究核验协议

## 信息源优先级

优先顺序如下：

1. 公司公告、交易所文件、监管文件、官方统计、央行/财政部门发布、公司投资者关系资料。
2. 交易所或可靠行情数据源，用于股价、成交额、市值、估值和流动性。
3. 电话会纪要、管理层演示材料、招股书、债券募集说明书、股东通函。
4. 权威财经媒体和数据服务商。
5. 券商研报、投行观点、一致预期和第三方数据库。
6. 社交媒体、市场传闻、论坛讨论只能作为情绪证据，不能当作事实。

## 时效规则

- 相关市场已收盘时，使用当天收盘数据。
- 市场交易中时，只能使用带明确时间戳的盘中数据。
- 如果无法取得实时数据，可以使用前一交易日收盘数据，但必须说明。
- 财报和公告必须识别最新年报、中报、季报、业绩预告、回购、分红、并购、减持、股权激励、重大合同等。
- 涉及政策和监管时，必须优先核验官方或高度权威来源，再做判断。

## 数据完整性

- 每个关键数字都要标注来源或来源类型。
- 如果不同来源数据冲突，说明差异，并优先采用原始公告或交易所数据。
- 不得编造财务数据、预测、机构观点、传闻、目标价或技术位。
- 区分历史指标和前瞻预测。预测年份必须写清楚，例如 `2026E PE`。
- 如果缺少预测数据，使用明确情景假设，不要给伪精确数值。

## 分析纪律

按“事实到推论”的链条表达：

1. 事实：已经核验了什么。
2. 解读：这对业绩、估值、流动性或情绪意味着什么。
3. 决策影响：对应什么操作或跟踪项。
4. 证伪条件：什么情况会推翻当前判断。

对热门概念，必须判断真实关联度：

- `强相关`：收入、利润、订单或客户暴露有实质贡献且可核验。
- `中相关`：有真实业务关联，但贡献还小或处于早期。
- `弱相关`：只是间接产业链映射或叙事映射。
- `蹭热点`：市场故事缺少可核验的业务支撑。

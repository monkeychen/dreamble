# x-reader 开发规范与约束

本项目旨在开发一款面向移动端的极简、丝滑的 HTML 与 Markdown 本地文档沉浸式阅读器，彻底解决微信等社交软件内此类文档无法直接优美预览、第三方打开链路冗长繁琐的痛点。

---

## 🛠️ 第一性原理：核心设计与交互原则

为了给用户（主要是创作者、程序员、知识工作者）提供极致的产品体验，我们必须严格遵循以下原则：

1. **用户体验至上（UX First）**
   - **零思考，极简链路**：用户在微信中点击“用其他应用打开”，应当能够一键直接在 `x-reader` 中渲染出最完美的排版，不要求用户做任何配置。
   - **系统承担复杂性**：微信环境的沙盒限制、中文排版的抖动、大文件渲染性能，全部在底层默默解决，前端界面必须保持丝滑。
   - **极致的视觉美学**：使用UI，支持精致的暗黑模式、高阶毛玻璃效果、流式卡片，以及类似 Typora 的高质感排版主题（如少数派风、GitHub 经典、沉浸木纹等）。

2. **渐进式展示（Progressive Disclosure）**
   - **沉浸化首屏**：打开文档默认 100% 区域为纯净阅读区，无杂乱按钮。
   - **交互触发细节**：轻触屏幕中部唤出顶部面包屑导航与底部状态条；点击右下角悬浮按钮展开文档大纲目录（Table of Contents）。

3. **离线与缓存第一**
   - 微信等应用内的沙盒文件会在 3-7 天内被系统强制清理。`x-reader` 必须在首次打开文件时，自动在本地沙盒存储一份轻量化的缓存备份，防止用户“下次想看时文件已失效”。

## 📂 项目结构规范

为了保持工作空间的整洁，新加入的代码和文件必须严格按照以下结构组织：

```
x-reader/
├── docs/                 # 项目文档与设计原稿
├── assets/               # 静态资源（图标、字体、演示文件）
├── src/                  # 核心源代码
│   ├── parser/           # HTML 与 Markdown 的极速解析与净化引擎
│   ├── theme/            # 排版样式与渲染主题系统 (CSS)
│   ├── components/       # 可复用轻量组件（如大纲抽屉、工具栏、手势监听）
│   └── main.ts           # 应用程序入口
├── temp/                 # 临时文件夹（用于调试抓取的 HTML/MD，需加入 .gitignore）
├── GEMINI.md             # 本协作规范（本文档）
└── package.json          # 依赖管理
```

- **清理机制**：`temp/` 目录下的所有临时文件（包括测试抓取的网页）仅用于本地调试，绝对禁止提交到 Git 仓库，每完成一个模块的测试必须及时清理。

## 🤖 AI 协作约定

- **技术决策先行**：在提交每一段代码或方案前，必须明确说明**「为什么」**以及**「对用户体验的影响」**，不准罗列代码实现细节。
- **验证驱动**：在通知用户成果之前，必须主动在本地环境跑通相关的单元测试、打包构建或排版渲染验证。
- **保持谦逊**：不使用自夸词汇，尊重事实，直击本质，发现设计瑕疵必须直言不讳。

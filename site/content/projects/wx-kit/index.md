## 解决什么问题

你在微信里收藏的文章，其实并不属于你：作者删文、平台限流、账号注销，它就没了。wx-kit 做一件朴素的事——**把公众号文章下载成实实在在的本地文件**，不依赖任何人的服务器继续存在。

另一个初衷：想把一批公众号文章喂给 AI 做分析整理时，不再一篇篇人肉复制粘贴。

## 核心功能

- **多格式下载**：一条文章链接 → 封面 / Markdown / 网页 / PDF / 元信息五种格式任选，图片全部本地化，图床挂了也不裂图
- **批量爬取**：搜公众号名字，扫码登录后按数量或日期范围批量下载历史文章；断点续传 + 去重，中断重跑自动跳过
- **文库管理**：下载的文章可搜索、排序、按公众号分组，应用内直接阅读（暖色杂志风排版）
- **类 RSS 订阅**：订阅公众号，应用开着时定时检查新文章，提醒或自动下载；每次检查白纸黑字记录，不做黑盒
- **给 AI 用的 CLI**：同一个程序带命令行模式，输出纯 JSON（stdout 走数据、stderr 走进度、退出码区分成败），AI agent 可以直接调用它下载、检索、导出文章

## 设计边界（明说在前）

- 订阅只在电脑开着、应用开着时运行，没有 7×24 后台服务器；登录态过期会明确提示，不装死
- 不装根证书、不改系统代理、不碰系统底层——宁可功能弱一点
- macOS 安装包未做签名公证，首次打开需在系统设置里手动「仍要打开」

## 技术与开源

纯 Node + Electron 单进程，无 Python 边车、无数据库（文件系统 + JSON 索引）。**Apache-2.0 开源**，全部代码、设计文档和每一程的复盘日志都在仓库里。

这个工具从需求到上线是与 AI 结对完成的，过程记录见文章：[我做了个小工具，把公众号文章从围墙里捞出来，顺便喂给AI](/posts/wx-kit-intro/)。

# 「旧杂志」主题改版 设计文档

- 日期：2026-07-17
- 状态：已确认（设计阶段）
- 背景：站主反馈现有主题为功能性默认样式、无设计感，确定改版方向为「暖色旧杂志」

## 1. 目标与原则

把站点视觉从「能看的网页」变成「一本暖色旧杂志」：米黄纸感、衬线印刷、安静克制。约束：

- **零客户端 JS 不变**（深色跟随系统用 `prefers-color-scheme`）
- **信息架构、URL、内容 schema 全部不动**——纯视觉层改版
- 不引入装饰图片素材，气质全靠字体、颜色、间距
- 备案页脚内容不动（样式可调）

## 2. 配色体系

CSS 变量两套，语义变量名沿用现有（bg/fg/muted/accent/border/card-bg），页面代码无需改颜色引用。

**纸色（日间，默认）**

| 变量 | 值 | 说明 |
|---|---|---|
| `--bg` | `#f6f0e3` | 米黄纸底 |
| `--fg` | `#3a3226` | 暖黑褐墨色（不用纯黑） |
| `--muted` | `#756750` | 次要文字，暖灰褐（对 --bg 对比度 4.85:1，达 WCAG AA） |
| `--accent` | `#9c4526` | 赭红（印章红）：链接、题花、选中态 |
| `--border` | `#e2d7c0` | 淡褐细线 |
| `--card-bg` | `#faf5ea` | 纸片（比纸底略浅） |

## 3. 字体与版式

- **思源宋体（Noto Serif SC）全站**：`@fontsource/noto-serif-sc` npm 包，unicode-range 分包自托管；权重 400（正文）/ 700（标题）/ 900（刊头）。`font-display: swap`。dist 体积增大十余 MB 属预期（浏览器按页面用字按需下载分包，首屏字体请求量目标 ≤ 300 KB）
- **引用块**用系统楷体栈：`"Kaiti SC", "STKaiti", "KaiTi", "Noto Serif SC", serif`（旧杂志引文排楷体惯例，零加载成本；Noto Serif SC 补充 Android/Linux）
- 正文 17px、行高 1.9、**栏宽 50rem**（最初按杂志窄栏设为 38rem，上线后站主反馈过窄，2026-07-17 经 44rem 最终定为 50rem——阅读舒适度优先于窄栏形式感）
- **文章首段首字下沉**：`article > :first-child::first-letter`（纯 CSS，仅文章详情页）
- meta 信息：小字号 + 字距加宽 + 中圆点分隔（如 `2026 · 6 · 17`）
- 链接样式：**仅文章/正文内容内**（`article` 范围）墨色文字 + 赭红下划线（`text-decoration-color`），hover 全赭红；目录列表、「全部 →」、卡片等功能性链接**无下划线**（杂志目录不画线）

## 4. 页面级杂志化

| 部位 | 处理 |
|---|---|
| 刊头（header） | 站名居中大字（900 重 + 字距），下一行小字导航，上下细线夹住；替代现有左右布局 |
| 首页简介 | 「刊首语」：居中、楷体、短句 |
| 首页/归档文章列表 | 目录页样式：标题与日期间点线引导（dot leaders，CSS 实现） |
| 归档页年份 | 大号章节数字 + 细线 |
| 文章页 | 标题居中大字；meta 居中小字；正文首字下沉 |
| 公众号标注 | 「首发于微信公众号」做成赭红细边框小「印章」样式 |
| 作品卡片 | 纸片质感：淡褐细边、无/极淡阴影、截图圆角；archived 改「已归档」角标（替代整体淡化） |
| 页脚 | 版权页样式：居中、细线、备案信息保留 |
| 区块分隔 | 花式分隔符 `❦` 替代秃线 |

## 5. 实现范围

- 重写 `src/styles/global.css`（配色 + 版式 + 上述组件样式）
- `src/layouts/Base.astro`：刊头/页脚结构调整 + 字体 CSS import
- 少量页面模板调整：`index.astro`（刊首语、目录列表）、`posts/index.astro`（年份章节）、`posts/[slug].astro`（印章标注类名）、`ProjectCard.astro`（角标）
- `package.json`：新增 `@fontsource/noto-serif-sc` 依赖
- 不改：内容 schema、helpers、导入/发布脚本、RSS/sitemap、robots、privacy.html

## 6. 验证方式

1. `npm run verify` 全过（内容约束、Astro 类型检查、单元测试和构建）
2. `npm run dev` 浏览器实测：浅/深两套 × 桌面/移动宽度 × 全部页面类型（首页/归档/文章/作品墙/作品详情/关于）
3. 构建产物检查：字体分包存在于 `dist/_astro/`；文章页 HTML 引用的字体 CSS 正常
4. 发布后线上抽查 + 文章页首屏字体请求量实测。**预算修订（2026-07-17 站主确认）**：全站衬线方案下长文页首访实测 ~1.4 MB（正文 400 权重占大头），原 ≤ 300 KB 目标对该方案不可达；决策为接受首访 ≤ 1.5 MB，配合 nginx 对 `/_astro/`（内容哈希文件名）下发 `Cache-Control: public, immutable` 强缓存，回访与翻页字体流量为 0。`font-display: swap` 保证首访文字零阻塞

## 7. 明确不做

- 手动深色切换开关（引入 JS，低频需求不破例）
- 装饰性图片/纹理素材
- 分页、搜索等功能性变更（与本次改版无关）
